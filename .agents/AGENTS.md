# GYEOL 프로젝트 개발 규칙

1. **디자인 재현 및 레이아웃 유지**: 제공된 이미지와 레이아웃 디자인을 거의 100% 완벽히 복제하여 구현한다.
2. **임시 이미지 경로 선지정**: 이미지 애셋은 임시 경로와 파일명으로 먼저 매핑한 후, 사용자가 교체할 수 있도록 제공한다.
3. **요청된 범위 외 변경 금지**: 오직 요청한 수정 및 변경 사항만 건드리고 다른 영역은 절대 임의로 수정하지 않는다.
4. **사전 승인 필수**: 코드 수정 또는 명령어 실행 전, 변경될 부분을 명확히 설명하고 사용자의 승인을 받는다.
5. **지정된 테마 컬러 및 서체 고수**: 정해진 색상 스키마와 서체 스타일만 활용하며, 개인적인 판단으로 색상이나 글꼴을 변형하지 않는다.
